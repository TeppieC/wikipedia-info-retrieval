Schema:
statement(id, subject, predicate, object)
our table named statement, and has four columns, namely id, subject, predicate, object
We will use the most intuitive approach to store the RDF data into relational database. There will be only one table in the database, the table will be of exactly 3 columns. The columns will be named as "subject", "predicate" and "object" respectively. All data will be stored as complete uri's, enclosing by <>. 


SQLite statement:
CREATE TABLE statement( id INT PRIMARY KEY, subject TEXT, predicate TEXT, object TEXT);

Explanation:
our strategy is to store each tuple of predicate, subject and object in one row. 
If the objects are string, they are stored in "" 
And we remove all the ^^ part and @ part at the end of the object
